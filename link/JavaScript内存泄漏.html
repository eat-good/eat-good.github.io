JavaScript内存泄漏

1.什么是内存泄漏

内存泄漏是指由于疏忽或错误造成程序未能释放已经不在使用的内存

内存泄漏并非是指内存 存在物理上的消失，而是应用程序分配某段内存后，由于设计错误，导致再释放该内存之前就失去了对改内存的控制。

简单理解：无用的内存还在占用，得不到释放和归还，严重的时候，无用的内存还会增加。导致系统卡顿甚至崩溃

2.内存生命周期

一般分为三个周期

1.分配期

分配所需要的内存，在前端JavaScript中内存是自动分配的

2.使用期
使用分配到的内存，在js中读写一个变量或者是对象的属性值

3.释放期：

不需要时将该内存释放和归还，js将会自动释放内存，（除闭包和部分bug外），内存泄漏也是出现在这个时期。内存没有被释放导致的。

3.javascript内存管理机制

JavaScript在创建变量的时候会自动分配内存，在不使用时，会”自动的释放“内存，释放的过程称为垃圾回收机制。

分配内存》》使用内存》》释放内存

1.内存分配

JavaScript定义变量就会自动分配内存，

2.内存的使用

在使用值得过程中，实际上就是对内存分配进行读取和写入得操作，操作得可能时一个变量或一个对象得属性值。

3.内存回收

前端的内存回收称为GC

内存泄漏也发生在这一步，js的内存回收机制虽然能回收大部分的内存，但是会出现无法回收的情况，需要手动清理。

4两种垃圾回收算法

1.引入技术垃圾回收机制

把对象是否不在需要简化定义为对象没有其他对象引用到他

如果没有引用则指向零引用，该对象将被垃圾回收机制回收

实际应用： 对象 =null // 对象就被回收了？？？？

2.标记清除法：

在变量进入执行环境的时候，标记为”进入环境“，当变量离开执行环境的时候，标记为”离开环境“，被标记进入环境的变量不回收，标记离开环境的则回收

环境可以理解为我们的作用域。但是全局作用的变量在页面关闭时候才销毁

5.JavaScript内存泄漏的一些场景

1.被遗忘的定时器，写在函数内部的，。

如果id为Node的元素从DOM中移除，该定时器仍会存在，同时，因为回调函数中包含对someResource的引用，定时器外面的someResource也不会被释放

2.意外的全局变量{eslint现在时会报语法错误的}

一个未声明变量的引用会在全局对象中创建一个新的变量。

全局变量的注意事项：

如果需要全局变量来存储很多数据，必须确保在使用过后将它设置为null或重新为他赋值。

常见的和全局变量相关的引发内存消耗增长的原因是缓存。缓存存储着可复用的数据。

为了让这种做法更高效，必须为缓存的容量规定一个上界。由于缓存不能被及时回收的缘故，缓存无限制地增长会导致很高的内存消耗。

3.被遗忘的事件监听器，使用是事件监听器后要移出监听器

解决方式：添加监听器addEventListener移除监听器：removeEventListener

4.闭包内的

闭包可以使变量常驻内存，但如果使用不当就会在成内存泄漏

5.没有清理DOM元素

6.被遗忘的ES6 set成员
7.被遗忘的订阅发布事件监听器

怎样避免内存泄漏

1）减少不必要的全局变量，或者生命周期较长的对象，及时对无用的数据进行垃圾回收；

2）注意程序逻辑，避免“死循环”之类的 ；

3）避免创建过多的对象 原则：不用了的东西要及时归还。
